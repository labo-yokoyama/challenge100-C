# C# WPF Markdown Previewer アプリケーション作成プロンプト

## 概要

Markdown記法で入力したテキストをリアルタイムでHTMLに変換し、プレビュー表示するWPFアプリケーションを作成します。

## プロジェクト構成

-   プロジェクト名: `MarkdownPreviewer`
-   フレームワーク: .NET 8.0 (またはそれ以降)
-   言語: C#
-   UIフレームワーク: WPF

## ステップバイステップの指示

### ステップ1: プロジェクトの作成とセットアップ

1.  `MarkdownPreviewer`という名前で新しいWPFアプリケーションプロジェクトを作成してください。
2.  NuGetパッケージマネージャーコンソールで以下のコマンドを実行し、必要なライブラリをインストールしてください。

    ```powershell
    Install-Package Markdig
    Install-Package `Microsoft.Web.WebView2`
    ```

### ステップ2: ViewModelの作成

MVVMパターンに従い、``MainViewModel.cs``という名前で新しいクラスを作成し、以下のコードを記述してください。

-   `ObservableObject`のような`INotifyPropertyChanged`を実装したベースクラスを継承します。
-   `MarkdownText` (string型): `TextBox`にバインドするプロパティ。
-   `HtmlContent` (string型): `WebView2`に表示するHTMLコンテンツを保持するプロパティ。
-   `MarkdownText`プロパティのセッターで、値が変更された際に``Markdig.Markdown.ToHtml`()`メソッドを使用して`HtmlContent`を更新するロジックを実装します。

### ステップ3: View (UI) の設計

``MainWindow.xaml``を以下のように編集してください。

1.  ウィンドウの`DataContext`に`MainViewModel`を設定します。
2.  `Grid`を使用してウィンドウを2つの列に分割します。
3.  左側の列に、`AcceptsReturn="True"`と`TextWrapping="Wrap"`を設定した`TextBox`を配置します。この`TextBox`の`Text`プロパティを、ViewModelの`MarkdownText`プロパティに`TwoWay`モードでバインドします。
4.  右側の列に`wv2:WebView2`コントロールを配置します。

### ステップ4: ViewとViewModelの連携

`WebView2`は直接HTMLコンテンツをバインドする機能が弱いため、添付プロパティ（Attached Property）を利用した`Behavior`を作成して、ViewModelの`HtmlContent`プロパティの変更を`WebView2`に通知する仕組みを実装します。

``WebViewBehavior.cs``というクラスを作成し、`Html`という添付プロパティを定義してください。このプロパティが変更されたときに、`WebView2`の`NavigateToString()`メソッドを呼び出してHTMLを更新するように実装します。

``MainWindow.xaml``の`WebView2`コントロールに、この添付プロパティを設定し、ViewModelの`HtmlContent`プロパティにバインドします。

### ステップ5: コードビハインドの更新

``MainWindow.xaml.cs``のコンストラクタで、`DataContext`に`MainViewModel`のインスタンスを設定してください。

以上のステップで、基本的なMarkdownプレビューワーが完成します。
