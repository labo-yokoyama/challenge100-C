# JSON形式のメモ帳アプリケーション作成プロンプト

## 概要
複数のメモを管理し、データをJSON形式で保存・読み込みができるWPFアプリケーションを作成します。

## プロジェクト構成
- プロジェクト名: JsonNotepad
- フレームワーク: .NET Framework 4.5.2
- 言語: C# 5.0以前
- UIフレームワーク: WPF, MVVM
- プロジェクトフォルダ: 07_JsonNotepad
- プロジェクトファイル: JsonNotepad.csproj
- 既定名前空間: JsonNotepad

## UI要件
- メインウィンドウを左右に分割する。
- 左側（メモ一覧パネル）
    - メモのタイトル一覧を表示するリストボックス。
    - 下部に「新規作成」「削除」ボタンを配置。
- 右側（編集パネル）
    - メモの「タイトル」を入力するTextBox。
    - メモの「内容」を入力するTextBox（複数行、スクロールバーあり）。
    - 右上に「保存」ボタンを配置。
- MainWindow.xaml.cs は InitializeComponent() のみで、コードビハインドは使用しない。

## ロジック要件
- NuGetパッケージ: `Newtonsoft.Json` を使用してJSONのシリアライズを行う。
- データ構造:
    - `NoteItem`クラス: ID、タイトル、内容、更新日時を保持。
- 保存機能:
    - 実行ファイルと同じフォルダに `notes.json` という名前でデータを保存。
    - アプリ起動時に `notes.json` があれば自動で読み込む。
- リアルタイム性:
    - リストで選択したメモの内容が右側に表示されること。

## コーディング規約
- フィールド先頭にアンダースコア（_）を付けない。
- すべてのクラス、プロパティ、メソッドに <summary> コメントを付ける。
- メソッドは、引数や戻り値が存在する場合のみ <param> や <returns> を記述する。
- 文字化けを防ぐため、ファイル入出力時は UTF-8 を明示的に指定する。
- C# 5.0 で使用できない文法（null条件演算子 `?.`、文字列補完 `$`、nameof等）は使用しない。

# JSON形式の個別ファイル管理メモ帳アプリケーション作成プロンプト

## 概要
メモごとに個別のJSONファイルとして保存・管理し、タイトルによる検索機能を持つWPFアプリケーションを作成します。

## プロジェクト構成
- プロジェクト名: JsonNotepad
- フレームワーク: .NET Framework 4.5.2
- 言語: C# 5.0以前

- プロジェクトファイル: JsonNotepad.csproj
- 既定名前空間: JsonNotepad

## UI要件
- メインウィンドウを左右に分割する（比率は 1 : 2）。
- 左側（管理パネル）
    - 上部に「検索」用のTextBoxを配置（プレースホルダー的に「検索...」と表示するか、ラベルを添える）。
    - 中央にメモのタイトル一覧を表示するListBoxを配置。検索文字に応じてリアルタイムに絞り込まれること。
    - 下部に「新規作成」「削除」ボタンを配置。
- 右側（編集パネル）
    - メモの「タイトル」を入力するTextBox（これがファイル名になります）。
    - メモの「内容」を入力するTextBox（複数行、自動垂直スクロールバーあり）。
    - 右上に「保存」ボタンを配置。
- MainWindow.xaml.cs は InitializeComponent() のみで、コードビハインドは使用しない。

## ロジック要件
- NuGetパッケージ: `Newtonsoft.Json` を使用する。
- 保存の仕組み:
    - 実行ファイルと同じフォルダに `notes` フォルダを作成し、その中に `[タイトル].json` という形式で個別に保存する。
    - アプリ起動時に `notes` フォルダが存在しなければ自動作成し、既存の全JSONファイルを読み込む。
- 検索機能:
    - 検索窓に入力した文字列がタイトルに含まれるメモのみをListBoxに表示する（CollectionView等の利用を想定）。
- データ構造:
    - `NoteItem`クラス: タイトル、内容、最終更新日時を保持。
- 制限事項の考慮:
    - ファイル名に使用できない文字がタイトルに使われた場合の簡単なサニタイズ、またはエラー回避を考慮する。

## コーディング規約
- フィールド先頭にアンダースコア（_）を付けない。
- すべてのクラス、プロパティ、メソッドに <summary> コメントを付ける。
- メソッドは、引数や戻り値が存在する場合のみ <param> や <returns> を記述する。
- 文字化けを防ぐため、ファイル入出力時は UTF-8 を明示的に指定する。
- C# 5.0 で使用できない文法（null条件演算子 `?.`、文字列補完 `$`、nameof等）は絶対に使用しない。
このプロンプトを使うことで、個別のJSONファイル管理と検索機能を備えたメモ帳アプリのコードを生成させることができます。

タイトルをファイル名にする場合、/ や : などの記号が入力されるとエラーになる可能性があるため、ロジック要件にその旨の注意書きを加えておきました。
